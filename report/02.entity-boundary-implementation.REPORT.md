## 2. 구현 계약별 현황

### 2.1 Entity — 불변식 (INV-*)

| ID | 규칙 | 구현 | 테스트 |
|----|------|------|--------|
| INV-SIZE | 4×4 형상 | `is_valid_magic_square`, `generate_magic_square` | `test_inv_size_*` |
| INV-POSITIVE-NATURAL | 셀 ≥ 1 | `_cells_are_positive_natural` | `test_inv_positive_natural_*` |
| INV-RANGE-3DIGIT | 1 ≤ 셀 ≤ 999 | `_cells_are_within_range`, `_generate_origin_cell` | `test_inv_range_3digit_*` |
| INV-UNIQUE | 16칸 중복 없음 | `_cells_are_unique` | `test_inv_unique_*` |
| INV-ROW-SUM | 행 합 = 34 | `row_sums`, `_line_sums_match_magic_constant` | `test_inv_row_sum_*` |
| INV-COL-SUM | 열 합 = 34 | `col_sums` | `test_inv_col_sum_*` |
| INV-DIAG-SUM | 주·부 대각선 합 = 34 | `diag_sums` | `test_inv_diag_sum_*` |

### 2.2 Entity — 수용 조건·API

| ID / API | 내용 | 구현 파일 |
|----------|------|-----------|
| AC-GEN-01 | INV-* 만족 4×4 행렬 생성 | `generate_magic_square()` |
| `row_sums` | 행 합 4개 반환 | `src/matrix.py` |
| `col_sums` | 열 합 4개 반환 | `src/matrix.py` |
| `diag_sums` | (주대각, 부대각) 합 반환 | `src/matrix.py` |
| `is_valid_magic_square` | INV-* 통합 검증 | `src/matrix.py` |

**생성 알고리즘 (AC-GEN-01)**

1. `time.time_ns()` 기반 random 시드
2. `[0][0]` 앵커 — `random.randint(1, 999)`
3. 기본 마방진의 회전·반전·행열 교환 변형 중 앵커 일치 템플릿 선택
4. 템플릿 기반 16셀 채움 (`random.random()` 호출 포함)

### 2.3 Boundary — 입력 계약 (E-*)

| ID | 입력 | 기대 동작 | 테스트 수 |
|----|------|-----------|-----------|
| E-MAT-SHAPE | 4×4가 아닌 배열 | `validate_magic_square` → `False` | 4 |
| E-CELL-ZERO | 0 포함 | `False` | 1 |
| E-CELL-NONINT | 문자·실수·None | `parse_matrix` → `ValueError` | 3 |
| E-CELL-NEG | 음수 | `False` | 1 |
| E-CELL-OVER-3DIGIT | 999 초과 | `False` | 1 |
| E-CELL-DUP | 중복 값 | `False` | 1 |

### 2.4 Boundary — 수용 조건 (AC)

| ID | 내용 | 상태 |
|----|------|------|
| AC-GEN-01 | 웹에서 「출력」 버튼 → 4×4 행렬 표시 (한 줄 4숫자 × 4줄) | **완료** |
| AC-VAL-01 | 사용자 4×4 입력 → 마방진 여부 확인 | 미구현 |
| AC-VAL-02 | 실패 규칙 구분 표시 (선택) | 미구현 |
